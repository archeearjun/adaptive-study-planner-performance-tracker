What the product does

Turns study planning into a feedback loop

Adaptive Daily Planning

Builds an ordered day plan from available time, deadlines, priority, backlog, and predicted recall probability.

SM-2 Reviews

Applies spaced repetition updates to easiness factor, repetition count, interval, and next review date.

Pomodoro Breakdown

Converts each planned study item into focused work blocks with configurable focus and break lengths.

Retention Prediction

Uses a custom logistic regression model to estimate whether a learner is likely to remember a topic.

Performance Analytics

Surfaces study streaks, completion rate, overdue reviews, neglected topics, and subject-level breakdowns.

Replanning After Progress

Logged work changes the same-day queue so completed topics are removed or shortened instead of being blindly repeated.

System design

Clean architecture with an explainable planning pipeline

01

Input Layer

Subjects, topics, deadlines, confidence, difficulty, estimated time, and review metadata enter through the JavaFX UI.

02

Persistence Layer

SQLite stores subjects, topics, sessions, review events, generated plans, Pomodoro blocks, and model training data.

03

Decision Layer

The scheduler computes a weighted score from urgency, importance, recall risk, difficulty, and backlog pressure.

04

Retention Layer

A custom logistic regression model predicts recall likelihood from behavior and review history, then feeds the scheduler.

05

Feedback Layer

Session logs update SM-2 state, add training examples, and improve future plans and analytics.

Distribution

Public sharing, packaging, and Vercel fit

What goes on Vercel

The repository root contains a static project site. Deploying this root directory on Vercel publishes the product overview, screenshots, and public links.

What stays desktop-only

The JavaFX application is distributed separately as a packaged Windows app image or GitHub release asset, not as a Vercel runtime.

How to package it

Run scripts/package-windows.ps1 to build a Windows app image and zip archive using jpackage.

How to deploy the site

Run scripts/deploy-vercel.ps1 -Production after authenticating with Vercel to publish this static site.

Run locally

Desktop app commands

Launch the JavaFX app

mvnw.cmd javafx:run

Refresh showcase screenshots

powershell -ExecutionPolicy Bypass -File scripts/export-screenshots.ps1

Build the Windows package

powershell -ExecutionPolicy Bypass -File scripts/package-windows.ps1

Deploy the static project site

powershell -ExecutionPolicy Bypass -File scripts/deploy-vercel.ps1 -Production

On macOS or Linux, use the Maven Wrapper directly for development. The Windows packaging script requires jpackage from JDK 17 and produces a zipped app image under dist/.
